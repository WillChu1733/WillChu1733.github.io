An analogy to describe JavaScript and its relationship to HTML and CSS

HTML, CSS and JavaScript can be compared to a Home. HTML provides the structure and foundation to create a building and CSS styles the interior, decorates the walls, decides the furniture etc. They both work together to transform a building into a house. The house becomes a Smart Home with the power of JavaScript, allowing interactivity between the house and the user. Examples such as wifi connectivity to the heat pump, connectivity to the light bulbs etc leads to an interactive environment between the user and the home.

An analogy of control flow and loops using examples from everyday life

Control flow is the order of statements, instructions or function calls which are executed using step by step instructions. For example, cooking pasta:

  1. Boil a pot of water
  2. Chop vegetables & prepare sauce
  3. Once water is boiling, add uncooked pasta
  4. Cook pasta for 8 minutes
  5. Drain pasta and add to sauce
  6. Stir and serve pasta

Loops allow us to repeat a series of instructions until a specific condition is met. For example making coffee for a group of people:

  1. determine the number of people you need to make coffee for
  2. Calculate amount of coffee beans required, based on number of people
  3. Divide coffee beans into equal portions
  4. For each person, make a cup of coffee using divided beans
  5. repeat step 4 until everyone is served

Describe what the DOM is and an example of how you might interact with it

The Document Object Model is a programming interface for web documents and allows JavaScript to be integrated with CSS and HTML to make the website interactive. The DOM is made up of nodes and objects (elements, attributes, text, content etc..) so that the programming languages (HTML, CSS, JS) can interact with the page. The DOM is represented by a Window Object and a Document Object. The window object is an interface that represents the browser and the document object is an interface that represents the root of the document itself and everything that lives in the document. We can use the DOM when we want to see or edit the structure of an object in real-time, without permanently saving these changes.

Explain the difference between accessing data from arrays and objects.

    An Array is an ordered collection of elements or values, starting at 0, incrementing by one for each element/value in the array. For example, if you have an array of numbers like [1, 2, 3, 4, 5], and you want to access the third element (which has an index of 2 because indexing starts at 0), you can use the notation myArray[2]. This will return the value 3.
    An Object is an unordered collection of key-value pairs. The values of an object can be accessed by its associated key. For example, if you have an object with key-value pairs like {name: "John", age: 25}, and you want to access the value associated with the key "name", you can use the notation myObject.name. This will return the value "John".
